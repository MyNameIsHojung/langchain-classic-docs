About this documentation

  • Built from local package source at langchain_classic.
  • langchain_classic is primarily a compatibility shim that re-exports symbols from langchain_core and langchain_community.
  • Dynamic imports emit LangChain deprecation warnings outside interactive environments.
  • For new projects, prefer the modern packages (langchain_core, langchain_community, langchain_text_splitters) and LCEL workflows.

How to use

  • Browse modules on the left to open their reference pages.
  • Each page summarizes exports, shims, and notes from the local package source.

Overview

  • langchain_classic preserves the legacy LangChain API surface while the project moves to split packages.
  • Core packages now live in: langchain_core, langchain_community, and langchain_text_splitters.
  • Most modules are dynamic import shims; a subset of classic implementations remain for backwards compatibility.

Use langchain_classic when you need legacy APIs or classic chains/agents that have not been migrated. For new projects, prefer LCEL and the modern packages.

Installation and requirements

pip install langchain-classic

Core requirements (from local metadata):

  • Python >= 3.10, < 4.0
  • langchain-core, langchain-text-splitters, langsmith
  • pydantic, pyyaml, requests, sqlalchemy
  • async-timeout (only for Python < 3.11)

Many integrations live in langchain-community and are imported dynamically.

Optional extras (provider bundles):

  • langchain-classic[openai], [anthropic], [aws], [google-genai], [google-vertexai]
  • [groq], [huggingface], [mistralai], [ollama], [deepseek]
  • [fireworks], [perplexity], [together], [xai]

Compatibility model

  • Shims use __getattr__ to import symbols lazily when accessed.
  • Deprecated imports emit LangChain warnings outside interactive environments.
  • Imports are restricted to langchain_core, langchain_community, and langchain_classic.
  • Some legacy features moved to langchain_experimental and raise ImportError.

Quickstart

Classic chain (legacy API)

from langchain_core.language_models import FakeListLLM
from langchain_core.prompts import PromptTemplate
from langchain_classic.chains import LLMChain

llm = FakeListLLM(responses=["hello"])
prompt = PromptTemplate.from_template("Say {thing}")
chain = LLMChain(llm=llm, prompt=prompt)

result = chain.invoke({"thing": "hi"})
print(result["text"])

LCEL equivalent (recommended)

from langchain_core.language_models import FakeListLLM
from langchain_core.output_parsers import StrOutputParser
from langchain_core.prompts import PromptTemplate

prompt = PromptTemplate.from_template("Say {thing}")
llm = FakeListLLM(responses=["hello"])
chain = prompt | llm | StrOutputParser()

print(chain.invoke({"thing": "hi"}))

Using the Hub

from langchain_classic import hub

prompt = hub.pull("owner/prompt_name")

Package layout (high level)

  • Top-level helpers: globals, hub, env, load, text_splitter, cache.
  • Classic implementations: agents, chains, memory, output_parsers, retrievers.
  • Community-backed shims: chat_models, embeddings, llms, tools, utilities, vectorstores.

Deprecations and migration

  • from langchain_classic import OpenAI → from langchain_community.llms import OpenAI
  • from langchain_classic import FAISS → from langchain_community.vectorstores import FAISS
  • from langchain_classic import PromptTemplate → from langchain_core.prompts import PromptTemplate
  • from langchain_classic.text_splitter import RecursiveCharacterTextSplitter → from langchain_text_splitters import RecursiveCharacterTextSplitter

LLMChain and classic agents remain for compatibility, but LCEL and LangGraph are recommended for new work.

Troubleshooting

  • ModuleNotFoundError: langchain_community → install langchain-community for integrations.
  • ImportError: moved to langchain_experimental → use langchain_experimental and sandbox tools.
  • Unexpected deprecation warnings → prefer direct imports from langchain_core / langchain_community.
