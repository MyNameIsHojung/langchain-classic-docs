Module path

langchain_classic.runnables

Summary

Non-core runnable utilities for the LangChain Expression Language (LCEL).

When to use

  • Route OpenAI function calls or load hub runnables.

Typical workflow

  • Use OpenAIFunctionsRouter or HubRunnable.

Migration notes

  • Prefer LCEL/core runnables for new workflows.

What it provides

  • Includes helper runnables that bind or route based on tool-call outputs.
  • Provides HubRunnable for loading a runnable from the LangChain Hub.
  • LCEL core interfaces live in langchain_core.runnables.

Notable exports

  • OpenAIFunctionsRouter (langchain_classic.runnables.openai_functions).
  • HubRunnable (langchain_classic.runnables.hub).

Notes and shims

  • The package __init__ only provides a module docstring; import concrete classes from their submodules.

Examples

Beginner

from langchain_classic.runnables.hub import HubRunnable

runnable = HubRunnable("owner/prompt_name")
runnable.invoke({"input": "hi"})

Intermediate

from langchain_classic.runnables.openai_functions import OpenAIFunctionsRouter
from langchain_core.runnables import RunnableLambda

router = OpenAIFunctionsRouter(
    runnables={
        "add": RunnableLambda(lambda x: x["a"] + x["b"]),
        "echo": RunnableLambda(lambda x: x),
    }
)

Advanced

from langchain_core.prompts import PromptTemplate
from langchain_core.output_parsers import StrOutputParser

chain = PromptTemplate.from_template("Say {thing}") | llm | StrOutputParser()
chain.invoke({"thing": "hello"})
