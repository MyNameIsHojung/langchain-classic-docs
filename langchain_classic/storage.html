Module path

langchain_classic.storage

Summary

Key-value stores and storage helpers for caching and persistence.

When to use

  • Cache embeddings or docs in KV stores.

Typical workflow

  • Use InMemoryStore or LocalFileStore/EncoderBackedStore.

Migration notes

  • Prefer langchain_core.stores for new code.

What it provides

  • In-memory stores for bytes and arbitrary values.
  • File-system backed byte store and encoder-backed store wrapper.
  • Helpers to build LangChain docstores and LC stores.
  • Dynamic imports for Redis and Upstash stores from langchain_community.

Notable exports

  • InMemoryStore, InMemoryByteStore, InvalidKeyException.
  • LocalFileStore, EncoderBackedStore.
  • create_kv_docstore, create_lc_store.
  • RedisStore, UpstashRedisStore, UpstashRedisByteStore (dynamic imports).

Notes and shims

  • Redis and Upstash stores require langchain-community and their client libraries.

Examples

Beginner

from langchain_classic.storage import InMemoryStore

store = InMemoryStore()
store.mset([("key", {"value": 1})])

Intermediate

from langchain_classic.storage import LocalFileStore

store = LocalFileStore("./cache_dir")

Advanced

from langchain_classic.storage import EncoderBackedStore, LocalFileStore
import json

base = LocalFileStore("./cache_dir")
store = EncoderBackedStore(
    base,
    key_encoder=lambda k: k,
    value_serializer=lambda v: json.dumps(v).encode(),
    value_deserializer=lambda b: json.loads(b.decode()),
)
